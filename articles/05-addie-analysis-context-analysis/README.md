## この回で一緒にたどるもの

- 授業設計で見落とされがちな **「教室の外側」の 5 つの制約**
  (時間・機材・組織・法制・倫理) の洗い出し方
- 「先生が言ったから AI を使った」と保護者に叱られた **架空の生徒相談** から
  出発し、どこに火種があったかを AI と一緒に地図化する手順
- 保護者・管理職・同僚に、**炎上させずに説明** するための言葉選び
- 「使うデータを最小に絞る」ための **具体的なチェックリスト**
- 少数派の学習者 (通信環境が弱い家庭、個別配慮を要する生徒) の視点で
  **見落としを反証** する方法

> [!TIP]
> この回は「一人で完結しない」回です。**「通読 → 校内で共有できる 1 枚を作る →
> 管理職・学年主任と 10 分だけ話す」** を目標にすると、次の一歩が軽くなります。

## 第1章 ── 相談カードの 1 行

7 月最初の月曜、朝のホームルーム後。佐藤先生の机の上に、
生徒 (架空: 2 年 A 組の一人) からの「相談カード」が置かれていました。

> 生徒 (架空: 2 年 A 組の一人) からの「相談カード」の内容を **教育目的に
> 再構成して要約** すると、こんな趣旨でした。
>
> > 「授業で AI を試すよう言われて ChatGPT で下書きを作ったところ、
> > 家庭で **『学校が個人情報を勝手に扱わせている』** と保護者に
> > 強く懸念された。」

※ 相談カードの原文はここには載せません。原文は教員のみが保管し、
分析にも公開にも使いません。

佐藤先生は言葉に詰まりました。第4回までは "教室の内側" の設計を進めてきました。
でも今回のつまずきは、**教室の外側 — 家庭・保護者・法制・校則** の領域です。

田中先生 (社会科) が声をかけます。

> 「うちも似た話が来たよ。**先に "外側" を整理しておかないと、
> 授業計画がどれだけ立派でも一発で崩れる** んだよね。」

第5回の課題は、この **「教室の外側の制約」を体系的に洗い出す** ことです。

---

## 第2章 ── なぜ「外側」を先に見るのか

授業設計の枠組みには、**授業内容そのもの** に焦点を当てるものが多いです。
でも AI 導入では、**技術と社会・組織の "結び目"** に問題が集中します。

こういう見方を、教育社会学では **Actor-Network Theory** (アクター・
ネットワーク理論) 的な視座と呼ぶことがあります。乱暴に要約すると、
「**教室で起きる出来事は、教師・生徒・教材・機材・校則・法律・保護者
などの "アクター" が絡まりあった結果として見える** 」という考え方です。
本稿では、ANT を厳密に適用するのではなく、その **「教室内外を一体で見る」
視点** だけを借ります。

さらに、生成 AI × 教育に特化した国際的なガイダンスとして、
**UNESCO の生成 AI 教育ガイダンス** があります
[[UNESCO 2023]](../../shared/references.md#unesco-2023)。
このガイダンスは、**「教育者は AI を導入する前に、必ず社会的・倫理的
影響を検討すべき」** と繰り返し述べています。

### 見落とされがちな 5 つのレイヤー

第5回では、次の 5 つを「制約レイヤー」として扱います。

| # | レイヤー | 例 |
| :-: | :-- | :-- |
| 1 | **時間** | 授業コマ、教員の準備時間、家庭学習時間 |
| 2 | **機材** | 端末、ネット、校内ネットワークのフィルタ、家庭端末の格差 |
| 3 | **組織** | 校則、学年方針、教科主任・管理職の合意、保護者会 |
| 4 | **法制** | 個人情報保護法、著作権法、学校 ICT 利用規程、AI サービスの利用規約 |
| 5 | **倫理** | 生徒プライバシー、公平性、AI 依存、教育者の説明責任 |

> [!IMPORTANT]
> これらは **「授業がうまくいかない理由の少なくない部分」** が潜む場所です。
> 教育内容の設計が完璧でも、5 レイヤーのどれか 1 つが崩れると、
> 単元は動きません。

---

## 第3章 ── 入力する情報 (プライバシー配慮)

制約分析に AI を使うときも、**個別の生徒名・家庭事情は入れません**。

- 校種・学年・学級規模・教科・単元名 (公開情報)
- 校内 ICT 環境の一般化された記述 (「端末 1 人 1 台 / 校内 Wi-Fi あり
  / 校外 Wi-Fi は自宅次第」)
- 校則の要点 (公開部分のみ) と学年方針 (匿名化)
- 想定される保護者の懸念 (今回の相談カードから得た "類型化された声")

> [!WARNING]
> 相談してきた生徒の名前・カード原本・家庭状況は投入しません。
> **AI に投入するのは "類型化された懸念"** です
> ("保護者から『学校が個人情報を勝手に扱わせている』という懸念が
> 出た" のように、個人が特定されない粒度に加工)。

また、この回では **校内ネットワークのフィルタや ICT 利用規程を回避する
方法は扱いません**。制約を超える必要があるときは、必ず管理職・情報担当・
教育委員会と正規のルートで相談してください。

## 第4章 ── プロンプト 01 の実行結果: 制約マップ

1 本目のプロンプト
`prompts/01-constraint-map.md` (準備中) は、
上記 5 レイヤーで制約を **列挙** させ、**優先度** を付けさせます。

### AI (Claude Opus 4.7) の実行結果 — 抜粋

> [!NOTE]
> 以下は 2026 年 7 月に **Claude Opus 4.7** に上のプロンプトを投入して
> 得られた実出力を、レイアウトのみ整えたものです。**入力は架空、
> 出力は実行結果** です。他モデル・他セッションでは、
> 挙げられる項目や順序が変わります。

#### 5 レイヤーの制約リスト (抜粋)

| # | レイヤー | 制約 | 深刻度 (仮) |
| - | :-- | :-- | :-: |
| 1 | 時間 | 教員が事前にプロンプトを校正する時間の確保 | 中 |
| 2 | 機材 | 家庭で端末を使えない生徒 (推定 2〜3 名) | 高 |
| 3 | 機材 | 校内ネットワークで外部 AI サービスにアクセス可か | 高 |
| 4 | 組織 | 学年主任・管理職への説明が済んでいない | 高 |
| 5 | 組織 | 学校 ICT 利用規程での「AI」の位置づけが未整備 | 高 |
| 6 | 法制 | 生徒作品の一部を AI に入れる場合の著作権と同意 | 高 |
| 7 | 法制 | 生徒個人情報 (氏名・成績) は原則入れない前提 | 中 (方針次第) |
| 8 | 倫理 | 使わない選択肢を保証する (opt-out 権) | 高 |
| 9 | 倫理 | 保護者への事前説明 | 高 |
| 10 | 倫理 | 少数派生徒 (LD, 通信環境弱) への配慮 | 高 |

#### 優先して着手すべき Top 3 (AI 助言)

1. **保護者・管理職への事前説明** — 今回の相談カードで最も可視化された火種
2. **家庭端末格差の把握と opt-out ルート** — 一部生徒の排除を防ぐ
3. **校内 ICT 規程の "AI" 節整備** — 教員個人の判断に依存させない

### 佐藤先生の反応

> 「授業のプランを立てる前に、"外側" の火種がこんなに並ぶんだ……。
> 特に **opt-out (使わない選択肢)** の視点は完全に抜けていた。
> 相談カードの生徒は、実はこれを言葉にしてくれていたのかも。」

## 第5章 ── プロンプト 02 の実行結果: 説明文の下書き

2 本目のプロンプト
`prompts/02-stakeholder-communication.md` (準備中) は、
**保護者向け・管理職向け・同僚向け** の 3 つの説明文を、
炎上リスクを抑えたトーンで生成します。

### AI の実行結果 — 抜粋 (保護者向け)

> **保護者の皆さまへ**
>
> 本校では、2 年 A 組 国語「羅生門」の学習で、生成 AI (対話型 AI) を
> **補助教材の一つ** として試験的に使うことを検討しています。
> 以下の点を、事前にご確認いただきたくお知らせします。
>
> 1. **必ず使うわけではありません。** ご家庭の方針で使わない選択肢
>    (opt-out) を保証します。使わなくても学習内容・評価に不利益は
>    生じません。
> 2. **生徒個人が特定される情報は入れません。** 氏名・成績・家庭事情は
>    投入しない前提で、教員が指導します。
> 3. **教員が最終判断します。** AI の出力は "候補" であり、
>    授業で扱うかどうかは教員が確認したうえで行います。
> 4. **ご意見・懸念があれば、いつでもご連絡ください。**

### 佐藤先生の反応

> 「私が書くと堅くなりがちだけど、AI の下書きは "保護者の不安に
> 先回りする語り口" が上手い。ただし、**最後の署名は私と管理職**。
> ここは絶対に AI に任せられない。」

管理職向け・同僚向けはさらに簡潔で、**「実施範囲・opt-out・
記録方法・撤退基準」** の 4 点セットで構成されていました。

## 第6章 ── プロンプト 03 の実行結果: データ最小化チェックリスト

3 本目のプロンプト
`prompts/03-data-minimization-checklist.md` (準備中) は、
**「AI にどこまで入れて、どこから入れないか」** を、
実運用のチェックリストとして生成します。

### AI の実行結果 — 抜粋

| 項目 | 入れてよい | 入れない | 判断保留 |
| :-- | :-: | :-: | :-: |
| 単元名・教科書名 | ✅ |  |  |
| 学級規模・学年 | ✅ |  |  |
| 学級プロファイル (クラスタ表現のみ) | ✅ |  |  |
| 生徒個人の氏名 |  | ❌ |  |
| 生徒の成績データ |  | ❌ |  |
| 生徒作文の原文 |  | ❌ |  |
| 保護者の懸念 (類型化) | ✅ |  |  |
| 校則の該当部分 (公開部分のみ) | ✅ |  |  |
| 家庭 ICT 環境の一般化された分布 | ✅ |  |  |
| 職員会議の議事メモ |  |  | ⚠️ |

#### AI の追加助言

- 「入れない」項目でも、**要点を教員が要約して入れる** ことは可
  (例: 「発言時の慎重さ = 高」というトレンド情報)
- 「判断保留」項目は、**管理職・情報担当と相談** してから判断する
- 迷ったら **入れない側に倒す** のが安全側の運用

### 反証プロンプトの結果 (抜粋)

- **少数派学習者の視点で見落としている制約:**
  - 通信環境が弱い家庭 (Wi-Fi なし、モバイル制限あり)
  - 個別の学習支援を受けている生徒 (プライバシーの取り扱いが特に厳格)
  - 帰国生徒・多言語背景の生徒 (AI 出力の言語選択の配慮)
- **見落としている法制:** 学校で扱う生徒作品の著作権 (生徒に権利あり)
- **見落としている倫理:** AI 使用の "強制感" (「使わないと不利」の
  無言の圧力)

## 第7章 ── 実務での歯止め

制約分析は "洗い出せば安心" ではありません。
**「洗い出したうえで、どこで止めるか」** が現場の判断です。
佐藤先生は次の 4 つの歯止めを先に決めました。

| 歯止め | 具体 |
| :-- | :-- |
| **1. opt-out を必ず保証** | 「使わない生徒」に不利益を出さない代替課題 |
| **2. 個人特定情報は "投入しない" が既定** | 迷ったら入れない |
| **3. 説明責任の署名は人間** | 保護者・管理職への文書は AI 下書き + 教員/管理職の署名 |
| **4. 撤退基準を先に決める** | 「相談カードが 3 件超えたら一旦停止」など数値で |

> [!TIP]
> 4 つ目の **「撤退基準を先に決める」** は特に重要です。うまくいっている
> ときは判断できても、火が着いたあとでは冷静に決められません。
> **やめる条件を先に決めておく** ことで、教員自身が守られます。

## 第8章 ── 佐藤先生のアクション (Week 0)

制約マップと 4 つの歯止めを持って、佐藤先生は次のことを実施しました。

1. **相談カードの生徒に返信** — 「相談してくれてありがとう。あなたの
   家庭で AI を使わない選択肢もちゃんと用意します」と、まず個別に返す
2. **学年主任・管理職へ 10 分ミーティング** — プロンプト 02 の管理職向け
   説明文を持参、承認をもらう
3. **家庭 ICT 環境の匿名アンケート** — 端末・通信環境・保護者の懸念を
   3 分で回答できる形で配布
4. **保護者向け説明プリント** — プロンプト 02 の下書きを教員・管理職で
   最終編集し、学級通信と一緒に配布
5. **校内 ICT 規程の "AI" 節** — 学年会と情報担当で来学期までに整備する
   議論を開始

これらは **単元開始の "前" に、Week 0 として** 実施されます。
第6回以降の Design フェーズは、この土台の上でしか成り立ちません。

## この回のまとめ

- 授業設計の外側 = **時間・機材・組織・法制・倫理** の 5 レイヤー
- AI に列挙させ、**優先度と Top 3** を出させることで議論の土台になる
- 保護者・管理職・同僚への **説明文の下書き** は AI が得意、
  **署名は人間**
- **データ最小化チェックリスト** で「入れない」を明文化する
- 少数派学習者視点で **反証** すると、見落としが浮かび上がる
- 撤退基準を **先に** 決めておくと、教員自身が守られる

## この回で自分の場合を試してみたい方へ

1. 自分の学級・単元で、**時間・機材・組織・法制・倫理** を 3 項目ずつ書く
2. `prompts/01-constraint-map.md` (準備中) を使い、
   AI に **見落としの追加** を出させる
3. 保護者向け説明文の下書きを
   `prompts/02-stakeholder-communication.md` (準備中)
   で試す (実配布前に必ず管理職確認)
4. データ最小化チェックリストを、
   `prompts/03-data-minimization-checklist.md` (準備中)
   で自校向けに調整

## よくある不安に、先にお答えします

**Q. 制約を洗い出したら "何もできない" となりませんか？**
A. その心配はよくわかります。制約の目的は **禁止すること** ではなく、
**「どこまで安全に踏み込めるか」を教員自身が理解すること** です。
洗い出した制約のうち、Week 0 で対応できる 3 つに絞れば、
残りは "並行して整えていく課題" として扱えます。

**Q. 保護者への説明で、逆に不安を煽ることになりませんか？**
A. これも起こり得ます。だからこそ **「使わない選択肢を先に示す」** ことが
決定的に大事です。「使う前提の説明」より「使う/使わないを選べる説明」の
ほうが、不安の総量は減ります。

**Q. 校内 ICT 規程に「AI」の記述がないと、そもそも使ってよいのか判断できません。**
A. これは第5回で扱いきれない大きな論点です。**まず「規程が未整備であること
自体を、管理職と共有する」** ところから始めてください。教員個人が判断を
背負うのではなく、組織で判断する構造にすることが目的です。

**Q. 相談カードの生徒に、教師として何を伝えるべきですか？**
A. これは第5回の物語で意図的に扱いました。**「相談してくれたこと自体が
価値がある」「使わない選択肢がある」「家族の懸念も正当だ」** の 3 点を、
評価と切り離して伝えることが出発点です。
