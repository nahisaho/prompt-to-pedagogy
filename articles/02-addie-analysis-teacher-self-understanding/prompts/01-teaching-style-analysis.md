# プロンプト 01: 教師の教え方分析 (16Personalities × 多重知能)

- **目的:** 教師の 16Personalities 結果と多重知能 (MI) テスト結果を統合的に読み、
  生成 AI と組み合わせたときに **強みが増幅される (Amplified) 教え方** と
  **リスクが高まる (At-Risk) 教え方** を析出する
- **想定モデル:** GPT-5 / Claude Opus 4.7 / Gemini 3 Pro（長文推論に強いモデル推奨）
- **教育学的根拠:**
  - Shulman (1986) の PCK — 教科内容と教授法は教師個人の中で統合される
  - ADDIE モデルの Analysis フェーズ (Branch 2009)
  - 生成 AI の「弱み平準化 × 強み増幅」効果 (Dell'Acqua et al. 2023)
- **想定ユーザー:** 小中高・高等教育の教員、教員研修担当、教育工学研究者
- **ライセンス:** [CC BY-NC 4.0](../../../LICENSE)

---

## 使い方

1. [16Personalities (日本語)](https://www.16personalities.com/ja) を受検し、
   結果ページの **タイプコード**・**5 側面のスコア**・**性格の強み/弱み記述** をコピーする
2. 多重知能テスト (例: [LiteracyNet MI Assessment](https://www.literacynet.org/mi/assessment/findyourstrengths.html))
   を受検し、**8 知能のスコア (0–5 など)** をコピーする
3. 下のテンプレートの `<...>` 部分を置換して LLM に投入する
4. 出力後、**反証プロンプト (Section 追記)** を必ず続けて実行する
5. 生成された表を [`../examples/sample-analysis-output.md`](../examples/sample-analysis-output.md) の様式で保存する

> [!WARNING]
> このプロンプトは **診断ツールの結果を絶対視しない** ようモデルに指示しています。
> 出力の語尾が「〜である」「〜と決まる」となっている場合は、モデルが指示に従っていない
> 兆候なので、再生成してください。

## プロンプト本体

````text
あなたは、インストラクショナルデザイン (ADDIE モデル) と教員研修の専門家であり、
生成 AI を教育現場に導入する際のリスク評価にも精通しています。

# タスク
以下に示す教師 <教師の識別子; 例: 教師A> の 2 種類の自己診断結果を読み、
次の 2 つを表形式で抽出してください。

1. **Amplified**: 生成 AI と組み合わせたときに、この教師の強みが増幅される教え方 (5 件)
2. **At-Risk**: 生成 AI と組み合わせたときに、この教師の強みが弱まる/教育的リスクが高まる教え方 (5 件)

# 前提と制約 (厳守)
- 16Personalities は「性格の固定分類」ではなく **「自己内省の触媒」** として扱う
- 多重知能テストは「学習者の知能に合わせる」ためではなく、
  **「教師自身の教授方略レパートリーの偏り」を可視化するため** に用いる
- 出力の各主張は **「〜傾向がある」「〜が示唆される」「〜の可能性がある」** の
  語彙で書き、決定論的な断定を避ける
- 各行に **「根拠 (どの診断項目・スコアに基づくか)」** 列を含める
- Amplified/At-Risk の各 5 件は、可能な範囲で **異なる教授場面** (授業設計 / 教材作成 /
  発問・対話 / 評価 / 学級経営 / 保護者連携 / 教員間協働) をカバーする
- 出力は日本語

# 入力: 16Personalities 結果
- タイプコード: <例: INFJ-A>
- Mind (Introverted vs Extraverted): <例: I 72%>
- Energy (Intuitive vs Observant): <例: N 66%>
- Nature (Thinking vs Feeling): <例: F 61%>
- Tactics (Judging vs Prospecting): <例: J 55%>
- Identity (Assertive vs Turbulent): <例: A 58%>
- 自由記述 (強み・弱み欄からの抜粋):
  <結果ページの記述をそのまま貼り付け>

# 入力: 多重知能 (MI) テスト結果
- 使用テスト名: <例: LiteracyNet Multiple Intelligences Assessment>
- 8 知能のスコア (満点 <例: 5>):
  - 言語的 (Linguistic): <スコア>
  - 論理数学的 (Logical-Mathematical): <スコア>
  - 空間的 (Spatial): <スコア>
  - 身体運動的 (Bodily-Kinesthetic): <スコア>
  - 音楽的 (Musical): <スコア>
  - 対人的 (Interpersonal): <スコア>
  - 内省的 (Intrapersonal): <スコア>
  - 博物学的 (Naturalist): <スコア>
- (任意) 実存的 (Existential): <スコア>

# 教師の文脈情報
- 校種・学年: <例: 高等学校 1〜3 年>
- 担当教科: <例: 国語 (現代文・古典)>
- 経験年数: <例: 8 年>
- 現在の AI 活用度 (0-5): <例: 2>
- 特に伸ばしたい/心配な領域 (自由記述): <例: 生徒の批判的思考を伸ばしたい>

# 出力形式
## サマリー (200 字以内)
教師の自己傾向を、診断結果を根拠にひとまとまりで記述。

## 表 1: Amplified — AI と組み合わせて強みが増幅される教え方
| # | 教え方 (具体) | AI の関わり方 | 期待される効果 | 根拠 (診断項目) |
| - | :------------ | :------------ | :------------- | :--------------- |
| 1 |               |               |                |                  |
| … |               |               |                |                  |

## 表 2: At-Risk — AI と組み合わせるとリスクが高まる教え方
| # | 教え方 (具体) | 想定されるリスク | 回避策 | 根拠 (診断項目) |
| - | :------------ | :--------------- | :----- | :--------------- |
| 1 |               |                  |        |                  |
| … |               |                  |        |                  |

## メタコメント
- 診断ツールの限界に触れつつ、上記結果をどう扱うべきかを 3 行で助言する
````

## 追加プロンプト: 反証 (Sycophancy 対策)

上記の出力を受け取ったあとに、必ず次を続けて投入してください。

```text
先ほどのあなたの分析に対して、以下を実行してください。

1. Amplified 表の各行について、「その主張が誤っている可能性」を 1 つずつ提示する
2. At-Risk 表の各行について、「そのリスクが過大評価である可能性」を 1 つずつ提示する
3. 診断結果からは根拠が薄いにもかかわらず、あなたが推測で書いた項目があれば列挙する
4. 全体として、この教師が **同僚に外部レビューを依頼すべき論点** を 3 つに絞って提示する

語尾は前回同様、決定論的な断定を避けること。
```

## 期待される出力の要件

- [ ] Amplified / At-Risk が各 5 件揃っている
- [ ] すべての行に「根拠 (診断項目)」が明記されている
- [ ] 語尾が「〜傾向がある」等になっている（断定していない）
- [ ] 反証プロンプトの出力で、少なくとも 1 件は「根拠が薄い」と自己申告している

## 評価ルーブリック（簡易）

| 観点 | 3: 優 | 2: 良 | 1: 要改善 |
| :--- | :---- | :---- | :-------- |
| 診断項目との対応 | すべての行に妥当な根拠 | 過半数に根拠 | 根拠不明の行が多数 |
| 教授場面の多様性 | 7 場面中 5 以上をカバー | 3〜4 場面 | 1〜2 場面に偏る |
| 断定回避         | すべて非断定 | 一部断定あり | 断定が大半 |
| リスクの実効性   | 具体的な回避策あり | 抽象的な回避策 | 回避策なし |
