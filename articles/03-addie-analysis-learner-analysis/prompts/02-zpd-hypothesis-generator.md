# プロンプト 02: ZPD 仮説ジェネレーター (Zone of Proximal Development)

- **目的:** クラスタごとに **発達の最近接領域 (ZPD)** の仮説を書き、単元設計の足場 (scaffolding) を計画する
- **想定モデル:** GPT-5 / Claude Opus 4.7 / Gemini 3 Pro
- **教育学的根拠:**
  - Vygotsky (1978) の ZPD — 「一人ではまだできないが、支援があればできる領域」
  - Wood, Bruner, Ross (1976) の scaffolding
  - Bloom's Taxonomy 改訂版 (Anderson & Krathwohl 2001)
- **想定ユーザー:** 教科担任、教員研修担当
- **ライセンス:** [CC BY-NC 4.0](../../../LICENSE)

---

## 使い方

1. プロンプト 01 の出力 (学級プロファイル) を用意する
2. 次に扱う **単元** と **その単元の到達目標** を決めておく
3. 下のテンプレートの `<...>` を置換して LLM に投入する
4. 反証プロンプトを続けて実行する
5. 出力を「単元指導計画のたたき台」に使う

---

## プロンプト本体

````text
あなたは、学習科学と Vygotsky の発達理論に精通した教員研修の専門家です。

# タスク
以下の学級プロファイルと単元情報から、
**クラスタごとに ZPD (発達の最近接領域) の仮説** を書き、
それぞれのクラスタに対する **足場 (scaffolding) の候補** を 2 件ずつ提示してください。

# 前提と制約 (厳守)
- ZPD はクラスタ全員の実態を代表するものではなく、あくまで仮説
- 「支援があればできること」は、その学年の発達段階から乖離しすぎない範囲で
- 足場は「教師が用意するもの」と「他の生徒との相互作用で得るもの」の双方を含める
- 語尾は「〜と想定される」「〜が可能かもしれない」を用いる
- 断定を避け、実際の授業での観察による修正を前提とする
- 出力は日本語

# 入力: 学級プロファイル (プロンプト 01 の出力)
<プロンプト 01 のクラスタ表を貼り付け>

# 入力: 次に扱う単元
- 単元名: <例: 「羅生門」(芥川龍之介)>
- コマ数: <例: 8 コマ>
- 到達目標 (現時点の粗い版): <例: テクストの多層的解釈と根拠提示>
- 教科上の位置づけ: <例: 現代文 B、二学期の主教材>

# 出力形式

## 単元全体の ZPD マップ (200 字以内)
学級全体として、この単元で "次の一歩" となる能力領域を記述する。

## クラスタごとの ZPD 仮説

| クラスタ | 一人でできること (現在) | 支援があればできること (ZPD) | 支援策の候補 (教師) | 支援策の候補 (他生徒との相互作用) |
| :-- | :-- | :-- | :-- | :-- |
| C1 |    |    |    |    |
| C2 |    |    |    |    |
| ... |    |    |    |    |

## 単元中の "足場を外していく" 段階案 (Fading)
- コマ 1-2: <濃い足場>
- コマ 3-5: <部分的な足場>
- コマ 6-8: <足場を外す>

## メタコメント
- クラスタ内の分散をどう扱うか (どの生徒が仮説から外れやすいか)
- 単元中に ZPD 仮説を修正するタイミング (推奨)
````

---

## 追加プロンプト: 反証

```text
先ほどの ZPD 仮説について、以下を実行してください。

1. 各クラスタの ZPD 仮説が過大 (発達段階に対して背伸びしすぎ) の可能性を挙げる
2. 各クラスタの ZPD 仮説が過小 (現状追認になっている) の可能性を挙げる
3. 「支援策」の中に、形式主義に陥りやすいものがあれば列挙する
4. 単元中に ZPD 仮説の修正が必要になるサインを 3 つ挙げる

語尾は決定論的な断定を避けること。
```

---

## 期待される出力の要件

- [ ] 全クラスタに ZPD 仮説と 2 種類の足場が明記されている
- [ ] Fading (足場を外していく) 段階が単元コマ数と対応している
- [ ] 反証出力で、少なくとも 1 クラスタは「過大」または「過小」の懸念が挙がっている
- [ ] クラスタ内の分散への言及がメタコメントにある

## 評価ルーブリック (簡易)

| 観点 | 3: 優 | 2: 良 | 1: 要改善 |
| :--- | :---- | :---- | :-------- |
| ZPD の妥当性 | 発達段階に整合 | 一部背伸び/追認 | 大半が乖離 |
| 足場の具体性 | 実装イメージ明確 | 抽象的 | 使えない |
| Fading 設計 | 段階が単元と対応 | 部分的対応 | 対応なし |
| 修正可能性 | サインが明示 | 抽象的 | 言及なし |
