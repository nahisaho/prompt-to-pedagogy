# プロンプト 01: 単元ドラフト自動生成エージェント

**用途:** 特別編第1回 シナリオ A (授業準備) のためのプロンプト。
単元名・学級プロファイル・指導観を渡すと、8 コマ分の
授業案ドラフトと反証を並列出力する。

**モデル:** ChatGPT / Claude / Copilot (推論モデル推奨: o1, o3, Claude Thinking)

**入力に含めないもの:**
- 生徒の実名・成績・面談メモの原本
- 個人特定できるエピソード

**入力に含めるもの:**
- 単元名、教科、学年、コマ数
- 学級プロファイル (匿名化済み、クラスタ表現)
- 教員自身の指導観 (要約)
- 学校書式・時間割の制約

---

## プロンプト本体

```
あなたは高校国語の授業設計を補助する非同期の同僚 (AI エージェント) です。
以下の情報を受け取り、8 コマ分の単元ドラフトを 2 つのパートで出力してください。

# 入力
- 単元名: {{単元名}}
- 教科・学年: {{教科}} / 高校 {{学年}} 年
- コマ数: 8 コマ (各 50 分)
- 学級プロファイル (匿名化):
  - クラスタ C1 (発言積極型) x 名
  - クラスタ C2 (書字重視型) x 名
  - クラスタ C3 (支援必要層) x 名
- 教員の指導観 (要約): {{指導観}}
- 制約: {{制約 (時間・機材・行事重なりなど)}}

# 出力

## パート 1: 単元ドラフト
以下の 3 段階で構成してください:
1. **到達させたい成果 (Desired Results):** 3 項目、うち 1 つは
   長く残ってほしい理解 (Enduring Understanding, EU)
2. **証拠 (Evidence):** 何を見取るか、コマごとの証拠を列挙
3. **学習計画 (Learning Plan):** 8 コマの流れ (各コマ 3 行:
   ねらい・主活動・見取り)

## パート 2: 反証
上記ドラフトに対して、以下の観点で弱点を 5 個以上挙げてください:
- 教科の本質からズレていないか
- 評価が偏っていないか
- 8 コマで完走可能か
- クラスタ C3 (支援必要層) が置き去りになっていないか
- 教員の指導観と齟齬がないか

# 制約
- 個人名・具体的エピソードは生成しない
- 「完璧な設計」ではなく「現場で修正可能な設計」を目指す
- 判断は教員が行う前提で、選択肢を複数提示する
```

---

## 使い方

1. `{{変数}}` を実際の情報で置き換える
2. 出力を読み、反証パートから 3 個以上の指摘を教員自身で選ぶ
3. 選んだ指摘を反映させ、ドラフト第 2 版を作る
4. 学年会・教科部会で 15 分で共有する

## 注意

- **AI が出したドラフトは "設計例" であり "完成品" ではない**
- 校内で使うときは、必ず学校の書式に合わせて再整形する
- クラスタ表現は、事前に学級プロファイル (本連載 第3回)
  の手順で作成しておくこと
