# プロンプト 01: 学級プロファイル合成 (Class Profile Synthesizer)

- **目的:** 匿名化された学級観察メモから、**教えるうえで押さえるべき学級傾向** を抽出する
- **想定モデル:** GPT-5 / Claude Opus 4.7 / Gemini 3 Pro (長文推論に強いモデル推奨)
- **教育学的根拠:**
  - Vygotsky (1978) の発達の最近接領域 (ZPD) — 集団の多様性を把握する第一歩
  - Universal Design for Learning (UDL) の学習者多様性の原則
  - ADDIE モデルの Analysis フェーズ (Branch 2009)
- **想定ユーザー:** 小中高・高等教育の教員、教員研修担当
- **ライセンス:** [CC BY-NC 4.0](../../../LICENSE)

---

## 使い方

1. 手元の観察記録・面談ノート・成績を **手元だけで** 読み、以下の手順で匿名化する:
   - 生徒に S1, S2, ... の記号を割り当てる
   - 生徒名・家庭背景・診断名・実発言・実作文・実成績を **一切書かない**
   - 記述は **傾向カテゴリ** (例: 「本文引用が少ない」「発言頻度低め」) に変換する
2. 似た傾向でクラスタリングし、**5〜7 クラスタ** に集約する
3. 下のテンプレートの `<...>` を置換して LLM に投入する
4. 出力後、**反証プロンプト (末尾)** を必ず続けて実行する
5. **小サンプルモード** (3 人分だけで試す) から始めることを推奨

> [!WARNING]
> このプロンプトは **匿名化された傾向情報のみ** を扱う設計です。
> 出力に個人特定が可能な記述が含まれていたら、入力側の匿名化が不十分な兆候です。
> 出力を破棄し、入力を作り直してください。

---

## プロンプト本体

````text
あなたは、インストラクショナルデザイン (ADDIE モデル) と学習者分析の専門家であり、
生徒プライバシーの倫理にも精通しています。

# タスク
以下に示す学級 <学級識別子; 例: 2 年 A 組> の **匿名化された観察メモ** を読み、
次の 4 つを作成してください。

1. **学級全体の傾向** (5 行以内、平叙文)
2. **教師が押さえるべき 5 つのポイント** (表: 学級傾向 / 授業設計への示唆 / 根拠)
3. **クラスタ間の相互依存の可能性** (どの群とどの群を組ませると学びが起きうるか)
4. **教師が扱わないほうがよい (このデータ範囲では判断できない) 論点**

# 前提と制約 (厳守)
- 入力は匿名化された **傾向メモ** である。個人特定・家庭背景の推測を一切しない
- 出力に「S1 は〜な性格」など個人へのラベリングを含めない
- クラスタは仮説であり、実際の生徒はクラスタをまたぐ・当てはまらないことがある
- 語尾は「〜傾向がある」「〜が示唆される」「〜の可能性がある」を用いる
- 学級規模が小さい (n < 10) 場合は、統計的一般化を控える旨を明記する
- 出力は日本語

# 入力: 学級の基礎情報
- 校種・学年: <例: 高校 2 年>
- 教科・週コマ数: <例: 国語 4 コマ/週>
- 学級規模: <例: 40 名 (男子 21 / 女子 19)>
- 到達度分布 (前学年末など): <例: 上位 25% / 中位 50% / 下位 25%>
- 教員が伸ばしたい領域: <例: 批判的思考の言語化>

# 入力: 匿名化されたクラスタメモ
## クラスタ C1 (n = <>)
- 特徴:
- 課題:

## クラスタ C2 (n = <>)
- 特徴:
- 課題:

(以下、C5〜C7 まで同様)

# 出力形式
## 学級全体の傾向 (5 行以内)

## 教師が押さえるべき 5 つのポイント
| # | 学級傾向 | 授業設計への示唆 | 根拠 (クラスタ) |
| - | :-- | :-- | :-- |
| 1 |    |    |    |

## クラスタ間の相互依存の可能性 (3 件)
- 組: C? × C? / 期待される学び / 想定される摩擦

## このデータ範囲では判断しないほうがよい論点 (3 件)
- 論点 / 判断を控える理由

## メタコメント (2〜3 行)
- 匿名化データの粒度が十分か、追加で必要な情報の種類を助言
````

## 小サンプルモード (3 人版)

40 人分の匿名化が難しい場合、まず 3 人だけで試すためのバリエーション:

````text
以下は、3 名の生徒 (S1, S2, S3) の匿名化された観察メモです。
このサンプルサイズでは学級全体の傾向は導けないことを前提に、
「3 人の間の違いから、教師がこの学級全体について
"何を確認しに行くべきか" の質問リスト」を 5 件生成してください。

# 入力
S1: 特徴 / 課題
S2: 特徴 / 課題
S3: 特徴 / 課題
````

## 追加プロンプト: 反証 (Sycophancy 対策)

上記の出力を受け取ったあと、必ず続けて投入してください。

```text
先ほどのあなたの分析について、以下を実行してください。

1. 5 つのポイントの各行について、「その示唆が誤っている可能性」を 1 つずつ挙げる
2. クラスタ分けそのものが不適切である可能性 (どの生徒がクラスタに収まりにくいか) を 2 件挙げる
3. 匿名化メモからは根拠が薄いのに、あなたが推測で書いた項目があれば列挙する
4. この学級プロファイルを 4 週間後にどう更新すべきかの手順を提案する

語尾は前回同様、決定論的な断定を避けること。
```

---

## 期待される出力の要件

- [ ] 5 つのポイントすべてに「根拠 (クラスタ)」が明記されている
- [ ] 語尾が「〜傾向がある」等になっている (断定していない)
- [ ] クラスタ間の相互依存が 3 件以上提示されている
- [ ] 「判断しない論点」に、家庭背景・診断情報・個別支援計画などが含まれている
- [ ] 反証プロンプトの出力で、「クラスタに収まらない生徒」への言及がある

## 評価ルーブリック (簡易)

| 観点 | 3: 優 | 2: 良 | 1: 要改善 |
| :--- | :---- | :---- | :-------- |
| 匿名化の徹底 | 個人特定不可能 | 一部推測含む | 個人特定可能 |
| クラスタ根拠 | すべて明示 | 過半数明示 | 根拠不明多数 |
| 断定回避 | すべて非断定 | 一部断定 | 断定が大半 |
| 実務適用性 | 授業設計に直結 | 抽象的な示唆 | 一般論のみ |
