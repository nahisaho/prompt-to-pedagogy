# プロンプト 03: UDL 多様性チェッカー (UDL Diversity Checker)

- **目的:** 学級プロファイルと単元計画を、UDL (Universal Design for Learning) の 3 原則で点検し、"取りこぼされている生徒" を明らかにする
- **想定モデル:** GPT-5 / Claude Opus 4.7 / Gemini 3 Pro
- **教育学的根拠:**
  - CAST の Universal Design for Learning (UDL) Guidelines
  - Engagement / Representation / Action & Expression の 3 原則
- **想定ユーザー:** 教科担任、特別支援コーディネーター、教員研修担当
- **ライセンス:** [CC BY-NC 4.0](../../../LICENSE)

---

## 使い方

1. プロンプト 01・02 の出力 (学級プロファイル / ZPD 仮説) を用意する
2. 現時点の単元計画の粗いアウトライン (発問・活動・評価) を書き出す
3. 下のテンプレートの `<...>` を置換して LLM に投入する
4. 反証プロンプトを続けて実行する
5. 出力の "取りこぼし候補" を単元計画にフィードバックする

> [!TIP]
> UDL は "多様性への配慮" ですが、**選択肢を無限に増やすことではありません**。
> 教師が扱いきれる範囲で、**2〜3 の実現可能な調整** を選ぶことが重要です。

---

## プロンプト本体

````text
あなたは、Universal Design for Learning (UDL) と包摂的な授業設計に精通した
教育コンサルタントです。

# タスク
以下の学級プロファイル・ZPD 仮説・単元計画を、UDL の 3 原則で点検し、
**"取りこぼされている生徒" の可能性** を明らかにしてください。
その上で、**教師が実装可能な 2〜3 の調整案** を提案してください。

# UDL の 3 原則 (前提)
- **Engagement (取り組み):** やる気の入り口の多様性
- **Representation (提示):** 情報の受け取り方の多様性
- **Action & Expression (表現):** アウトプットの仕方の多様性

# 前提と制約 (厳守)
- 「調整案」は、教師の負担を過剰に増やさない範囲に限る
- 「すべての生徒に対して全ての選択肢」ではなく、"必要な生徒に必要な調整" を志向
- 特別支援教育としてではなく、**全員のための設計** として提案する
- 語尾は非断定 (「〜の可能性」「〜と想定される」)
- 出力は日本語

# 入力: 学級プロファイル (プロンプト 01)
<プロンプト 01 の要点を貼り付け>

# 入力: ZPD 仮説 (プロンプト 02)
<プロンプト 02 のクラスタ表を貼り付け>

# 入力: 現時点の単元計画 (粗い版)
- 単元名:
- 主な発問:
- 主な活動:
- 評価方法:

# 出力形式

### Engagement の穴
- 具体的な穴 (現状 → 想定される影響)
- 影響を受けやすいクラスタ:

### Representation の穴
- 同上

### Action & Expression の穴
- 同上

## 実装可能な調整案 (2〜3 件、優先順)
| # | 調整案 | 対象クラスタ | 想定される効果 | 教師の負担 (低/中/高) |
| - | :--    | :--          | :--            | :--                    |
| 1 |        |              |                |                        |

## 調整によって新たに生まれるリスク (2 件)
- リスク / 対処案

## メタコメント (2〜3 行)
- 「全員に全選択肢」を避けるための判断基準
- UDL 適用の限界 (UDL だけでは解決しない課題)
````

## 追加プロンプト: 反証

```text
先ほどの UDL 点検について、以下を実行してください。

1. 「取りこぼし候補」の各項について、それが過剰指摘である可能性を挙げる
2. 調整案の 1 つを実装したときに **生じうる不公平** (別の生徒にとっての不利益) を挙げる
3. UDL の枠組み自体が扱えない論点 (例: 学級運営の根本設計) があれば列挙する
4. 「まず 1 つだけ実装するなら」の推奨と、その理由を述べる

語尾は決定論的な断定を避けること。
```

---

## 期待される出力の要件

- [ ] 3 原則それぞれに具体的な穴が挙がっている
- [ ] 調整案に「教師の負担」が明記されている
- [ ] 「新たに生まれるリスク」が 2 件以上挙がっている
- [ ] 反証出力に「不公平の可能性」への言及がある

## 評価ルーブリック (簡易)

| 観点 | 3: 優 | 2: 良 | 1: 要改善 |
| :--- | :---- | :---- | :-------- |
| 3 原則の網羅 | 3 原則すべてに具体的指摘 | 2 原則 | 1 原則以下 |
| 実装可能性 | 教師の負担明示 | 抽象的 | 過負荷 |
| リスク開示 | 新リスク 2 件以上 | 1 件 | 言及なし |
| 反証の質 | 不公平/限界に言及 | 部分的 | 追従的 |
