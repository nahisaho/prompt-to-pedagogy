# プロンプト 02: 90 日間 AI 統合計画の生成

- **目的:** プロンプト 01 で得られた **Amplified / At-Risk リスト** をもとに、
  教師個人が実施可能な **90 日間の生成 AI 統合行動計画** を作成する
- **想定モデル:** GPT-5 / Claude Opus 4.7 / Gemini 3 Pro
- **教育学的根拠:**
  - Kolb (1984) の経験学習サイクル (経験→省察→概念化→試行)
  - Guskey (2002) の教員の変化モデル (実践→成果→信念変容)
  - ADDIE の Analysis フェーズから Design フェーズへの橋渡し
- **想定ユーザー:** 個々の教員、教員研修ファシリテーター
- **ライセンス:** [CC BY-NC 4.0](../../../LICENSE)

---

## 使い方

1. プロンプト 01 の出力 (Amplified 表 / At-Risk 表 / 反証出力) を手元に用意する
2. 下のテンプレートの `<...>` を置換して LLM に投入する
3. 出力を印刷またはカレンダーツールに転記し、**週次で振り返り** を行う
4. 30 日目・60 日目に **プロンプト 01 を再実行** して、自己プロファイルの更新有無を確認する

# 制約
- 90 日を 3 スプリント (0–30 / 31–60 / 61–90 日) に分割する
- 各スプリントは **経験学習の 4 段階** (Concrete Experience → Reflective Observation →
  Abstract Conceptualization → Active Experimentation) を 1 周させる
- 各スプリントには次を必ず含める:
  - Amplified から 1 つを **試行**
  - At-Risk から 1 つを **回避策とセットで検証**
  - 根拠が薄い項目のうち 1 つを **同僚 1 名にレビュー依頼**
- 各アクションに **所要時間の目安 (分/週)** と **成功指標 (観察可能な形)** を付す
- 教員の勤務時間を圧迫しないよう、**1 週あたり合計 90 分以内** に収める
- 出力は日本語 Markdown
